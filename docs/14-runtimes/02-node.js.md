# Node.js

## Running JavaScript Outside the Browser

Before Node.js, JavaScript could only run in browsers. Today, [Node.js](https://nodejs.org/en/download/) lets you execute JavaScript on your machine like any other scripting language.

If you've worked with Python, you're familiar with:

```bash
python main.py
```

With Node.js installed, you can do the same with JavaScript:

```bash
node main.js
```

Of course, you can still run JavaScript in your browser using the developer tools console, but Node.js opened an entirely new world for server-side and command-line JavaScript development.

## Managing Node Versions with NVM

As Node.js evolves, you may need different versions for different projects. This is where [**Node Version Manager (NVM)**](https://github.com/nvm-sh/nvm) becomes invaluable.

NVM allows you to:

- **Install multiple versions of Node** — Keep several versions on your machine simultaneously
- **Switch versions easily** — Change your active Node version with a single command
- **Manage per-project configurations** — Lock specific Node versions to individual projects
- **Keep installations clean** — Avoid conflicts and compatibility issues

Think of NVM as the JavaScript equivalent of `pyenv` for Python—it solves the same version management problem elegantly.

## Getting Started

Once you have Node.js and npm (Node Package Manager) installed, you can:

- Run JavaScript files directly with `node`
- Install and manage dependencies
- Build, test, and deploy applications
- Create command-line tools and server applications

Node.js has become the foundation of modern JavaScript development.
