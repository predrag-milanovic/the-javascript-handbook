# Comparisons

How JavaScript decides whether things are equal, how multiple conditions combine, and how “empty” or missing values are treated in conditional logic.

## Lessons in this chapter

- [01 – Conditionals](./01-conditionals.md) – `if`, `else if`, and `else` blocks that run code only when a condition is met.
- [02 – Comparison operators](./02-comparison-operators.md) – `<`, `>`, `<=`, `>=` and why `===` / `!==` are preferred over `==` / `!=`.
- [03 – Logical operators](./03-logical-operators.md) – `&&`, `||`, and `!` for combining and inverting boolean expressions.
- [04 – Switch](./04-switch.md) – `switch` statements as a cleaner alternative to long `if/else if` chains.
- [05 – Ternary operator](./05-ternary-operator.md) – the `condition ? valueIfTrue : valueIfFalse` shortcut for simple, inline decisions.
- [06 – Truthy and falsy](./06-truthy-and-falsy.md) – which values count as “on” or “off” in JavaScript’s boolean contexts.
- [07 – Nullish coalescing](./07-nullish-coalescing.md) – using `??` to provide safe defaults only when values are `null` or `undefined`.
