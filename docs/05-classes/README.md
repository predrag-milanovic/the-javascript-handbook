# Classes

How JavaScript uses `class` syntax to model reusable blueprints for objects, with constructors, public and private fields, static members, inheritance, and the `super` keyword.

## Lessons in this chapter

- [01 – Classes](./01-classes.md) – declaring classes, using `constructor`, and creating instances with `new` to organize related data and behavior.
- [02 – Private properties](./02-private-properties.md) – using `#` fields to hide internal state and encapsulate data inside a class.
- [03 – Static methods](./03-static-methods.md) – attaching properties and methods to the class itself instead of individual instances.
- [04 – Getters and setters](./04-getters-and-setters.md) – exposing computed or validated values that still look like plain properties.
- [05 – Inheritance](./05-inheritance.md) – extending a base class with `extends` to reuse behavior and override methods in subclasses.
- [06 – Super](./06-super.md) – calling parent constructors and methods with `super()` and `super.methodName()` from within a subclass.
