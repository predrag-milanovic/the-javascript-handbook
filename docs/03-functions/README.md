# Functions

How to define reusable chunks of code, pass data in and out of them, control what they can see, and use them as building blocks for more complex behavior.

## Lessons in this chapter

- [01 – Functions](./01-functions.md) – basic function declarations, calling functions, and returning values.
- [02 – Function hoisting](./02-function-hoisting.md) – how function declarations are hoisted, and why some functions can be called before they appear in the code.
- [03 – Multiple return values](./03-multiple-return-values.md) – patterns for returning more than one piece of data from a function.
- [04 – Functions as values](./04-functions-as-values.md) – treating functions like any other value: assigning them to variables, passing them around, and returning them from other functions.
- [05 – Scope](./05-scope.md) – what variables a function can “see”, and how nested functions create inner scopes.
- [06 – Anonymous functions](./06-anonymous-functions.md) – using functions without names, especially as callbacks and inline helpers.
- [07 – Default parameters](./07-default-parameters.md) – giving function parameters default values so callers can omit them.
- [08 – Passing by value](./08-passing-by-value.md) – how primitive values are copied when passed into functions, and why changes to parameters don’t affect the original variables.
- [09 – Immediate invocation](./09-immediate-invocation.md) – Immediately Invoked Function Expressions (IIFEs) that define and run a function in one step to create a small, self-contained scope.
